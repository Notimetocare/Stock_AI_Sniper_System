# 📈 股票型態輔助監控系統 (Stock Pattern Monitoring System)

> **專案概述**：
> 這是一個個人開發的實驗性專案，主要需求是為了解決「人工盯盤耗時」的問題。
> **本系統的開發過程深度依賴 AI 輔助 (LLM 協作)**。專案重點在於展示我如何透過與 AI 的溝通，從零開始整合資料爬蟲、多進程運算、特徵處理、以及 LINE Bot 後端開發，最終完成一套能穩定運作的雙層架構系統。

---

## 🏗️ 系統架構概念

本系統主要分為兩個模組，將「耗時的資料運算」與「輕量的訊息推播」分開處理：

### 📂 [1_Local_AI_Engine (本地端運算)](./1_Local_AI_Engine)
負責處理繁重的歷史資料下載與型態篩選。
* **資料獲取**：串接 `yfinance` 獲取台美股每日 K 線資料。
* **型態篩選**：將特定的 K 線型態 (如 S-L-R 結構) 邏輯化，篩選出符合條件的標的。
* **效能處理**：因應大量標的掃描，透過 `multiprocessing` 模組進行多進程處理以節省時間。

### 📂 [2_Cloud_Line_Bot (雲端推播與介面)](./2_Cloud_Line_Bot)
負責接收本地端的運算結果，並與使用者介面互動。
* **LINE 推播**：串接 LINE Messaging API，每日定時推播篩選結果。
* **後端框架**：使用 `Quart` 非同步框架，解決 LINE Webhook 回應超時 (Timeout) 的問題。
* **簡易網頁介面**：透過 LIFF 建立簡易的個人追蹤清單與狀態管理介面。

## 🛠️ 開發過程中的技術紀錄 (AI 協作解 Bug 日誌)

在與 AI 協作開發的過程中，遇到了許多系統整合的坑，以下是主要的解決紀錄：

1. **Python 效能瓶頸 (CPU-Bound)**
   * **問題**：初期掃描全市場耗時過長。
   * **解法**：學習並導入 `ProcessPoolExecutor`，將任務分配至多核心執行，大幅縮短運算時間。
2. **Matplotlib 背景繪圖導致記憶體耗盡**
   * **問題**：程式在背景產圖時容易引發伺服器卡頓或閃退。
   * **解法**：查閱文件後，強制設定 `matplotlib.use('Agg')` 關閉 GUI 渲染，並確實釋放資源。
3. **Webhook Timeout 處理**
   * **問題**：LINE 要求在幾秒內回覆，但程式有時處理過久導致失敗。
   * **解法**：放棄傳統 Flask，改用支援非同步的 `Quart`；並學會使用 `subprocess.Popen` 將耗時任務放到背景執行，先回覆 LINE 伺服器。
4. **圖片快取 (Cache) 問題**
   * **問題**：LINE APP 會快取同網址的舊圖片，導致圖表不更新。
   * **解法**：在圖片網址後方動態加入時間戳記 (`?v={timestamp}`) 來強制更新圖片。
